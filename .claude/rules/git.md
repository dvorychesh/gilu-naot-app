# Git

## Commit Messages – Conventional Commits
- `feat:` פיצ'ר חדש
- `fix:` תיקון באג
- `chore:` תחזוקה (deps, config)
- `refactor:` שיפור קוד ללא שינוי behavior
- `test:` הוספת/עדכון טסטים
- `docs:` תיעוד בלבד

## Branching
- `main` → production only
- `feat/[name]` → פיצ'רים
- `fix/[name]` → תיקוני באגים

## PR Guidelines
- PR קטן וממוקד – שינוי אחד בלבד
- חובה: tests ירוקים לפני merge
- אסור לעולם force-push ל-main
